## Candidate Experiences

### Candidate A — Software Engineer L3 (New grad, full loop)
"Two phone screens before onsite — both live coding on a shared doc, no autocomplete, which felt
strange at first. First problem was a balanced binary tree check, second was a trie
implementation. The onsite had 4 rounds: two more algorithm rounds (longest increasing
subsequence, and a matrix BFS problem), one Googleyness round full of 'tell me about a time...'
questions, and one round that was surprisingly deep on my resume project. The interviewers kept
asking me to think out loud even when I was stuck, which actually helped since they'd nudge me in
the right direction rather than just watching silently."

### Candidate B — Software Engineer L4 (Lateral, GCP team)
"Phone screen was a hard DP problem (word break). Onsite included a full system design round —
design a distributed key-value store — where they pushed hard on consistency vs availability
trade-offs. Also had two more coding rounds and a Googleyness round. What stood out was how
much they cared about clean, readable code even under time pressure — one interviewer explicitly
said sloppy variable names would count against me even if the logic was correct."

### Candidate C — Site Reliability Engineer (Lateral)
"Different flavor from pure SWE — more systems and troubleshooting-oriented questions alongside
coding. One round was purely about diagnosing a hypothetical production incident step by step.
Coding rounds were still algorithm-heavy though, including a min-heap implementation from
scratch. The hiring committee process took a couple of extra weeks after the onsite before I
heard back, which recruiters had warned me about in advance."

### Candidate D — Software Engineer L3 (New grad, project deep-dive)
"They spent almost 20 minutes of one round just on my resume project — a resume analyzer app
using LangChain and Azure OpenAI. Questions included why I used a vector database for retrieval
instead of just stuffing everything into the prompt, and how I'd scale the system if usage grew
10x. It reinforced that Google interviewers treat resume projects as a real technical
conversation, not just a formality."
