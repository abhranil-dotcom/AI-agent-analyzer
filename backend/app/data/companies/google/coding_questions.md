## Coding Questions

Google's coding bar is Medium-to-Hard, with unusually strong emphasis on clean code, precise
complexity analysis, and clear verbal reasoning while solving — interviewers are explicitly
trained to evaluate *how* you think, not just whether you reach the right answer.

### Find the longest increasing subsequence in an array.
- **Difficulty**: Medium-Hard
- **Topic**: Dynamic Programming
- Expect a follow-up pushing from the O(n²) DP solution to the O(n log n) patience-sorting
  approach.

### Given a binary tree, determine if it's height-balanced.
- **Difficulty**: Medium
- **Topic**: Trees / Recursion
- Tests recursive thinking and the ability to compute multiple properties in one traversal.

### Implement a trie (prefix tree) supporting insert and search.
- **Difficulty**: Medium
- **Topic**: Trees / Design
- Common in questions related to autocomplete-style features.

### Find the median of two sorted arrays.
- **Difficulty**: Hard
- **Topic**: Arrays / Binary Search
- A well-known hard problem testing binary search applied in a non-obvious way.

### Word Break — determine if a string can be segmented into dictionary words.
- **Difficulty**: Medium-Hard
- **Topic**: Dynamic Programming / Strings
- Expect discussion of both the DP approach and its complexity.

### Design and implement a min-heap from scratch (without a built-in library).
- **Difficulty**: Medium-Hard
- **Topic**: Heaps / Design
- Tests genuine data-structure understanding beyond just calling a library function.

### Given a matrix, find the shortest path from top-left to bottom-right avoiding obstacles.
- **Difficulty**: Medium
- **Topic**: Graphs / BFS / Matrices
- Common matrix-as-graph traversal problem.

### Clone a graph.
- **Difficulty**: Medium
- **Topic**: Graphs / DFS-BFS
- Tests careful handling of visited-node tracking to avoid infinite loops.

### Find all valid combinations of parentheses for n pairs.
- **Difficulty**: Medium
- **Topic**: Backtracking
- Classic backtracking problem with a clean recursive structure.

### Given a large text corpus, design an efficient algorithm to find the top-K most frequent words.
- **Difficulty**: Hard
- **Topic**: Heaps / HashMaps / Design at Scale
- Often extended into a system-design-flavored discussion of doing this over distributed data.
