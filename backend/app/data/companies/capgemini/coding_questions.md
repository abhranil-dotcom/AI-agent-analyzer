## Coding Questions

Capgemini's coding bar is Easy-to-Medium for general fresher hiring, with Medium-difficulty
questions more common for digital engineering and cloud-track lateral hires.

### Find duplicate elements in an array.
- **Difficulty**: Easy
- **Topic**: Arrays / HashMaps
- Expect a follow-up on optimizing from O(n²) brute force to an O(n) HashSet-based approach.

### Check if two strings are anagrams.
- **Difficulty**: Easy
- **Topic**: Strings
- Frequency-count or sorting-based approaches are both accepted.

### Implement a queue using two stacks.
- **Difficulty**: Medium
- **Topic**: Stacks / Queues
- A recurring Capgemini favorite that tests understanding of both data structures.

### Find the maximum sum subarray (Kadane's algorithm).
- **Difficulty**: Medium
- **Topic**: Arrays / Dynamic Programming
- Expect a follow-up asking you to trace through the algorithm on a sample array.

### Write a function to check for balanced parentheses in a string.
- **Difficulty**: Easy-Medium
- **Topic**: Stacks
- Classic stack-based validation problem.

### Merge two sorted arrays into one sorted array.
- **Difficulty**: Easy
- **Topic**: Arrays / Merging
- Expect a follow-up on doing it in-place vs using extra space.

### Find the first non-repeating character in a string.
- **Difficulty**: Easy
- **Topic**: Strings / HashMaps
- Common warm-up question with a clean HashMap-based solution.

### Implement a basic REST endpoint (pseudocode) that returns a list of resources with pagination.
- **Difficulty**: Medium
- **Topic**: API Design
- Reserved for digital-engineering-track candidates; tests practical API design thinking, not
  just algorithms.

### Write a SQL query to find the top 3 highest-paid employees per department.
- **Difficulty**: Medium
- **Topic**: SQL
- Expect a window-function-based approach (`RANK()`/`DENSE_RANK()` with `PARTITION BY`).

### Reverse words in a sentence without using built-in split/reverse utilities.
- **Difficulty**: Easy-Medium
- **Topic**: Strings
- Tests manual string manipulation and edge-case handling (multiple spaces, leading/trailing
  spaces).
