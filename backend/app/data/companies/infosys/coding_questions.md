## Coding Questions

Infosys's general fresher track skews Easy-to-Medium, similar to TCS, but the Power Programmer
track raises the bar noticeably to Medium-Hard with a strong DSA focus.

### Check if a number is an Armstrong number.
- **Difficulty**: Easy
- **Topic**: Math
- A recurring Infosys favorite for the general fresher assessment.

### Find the sum of digits of a number recursively.
- **Difficulty**: Easy
- **Topic**: Recursion
- Tests basic recursion and base-case handling.

### Reverse a linked list.
- **Difficulty**: Medium
- **Topic**: Linked Lists
- Common in technical interviews; expect a follow-up on doing it iteratively vs recursively.

### Find all pairs in an array that sum to a target value.
- **Difficulty**: Easy-Medium
- **Topic**: Arrays / HashMaps
- A HashMap-based O(n) approach is the expected optimal solution.

### Implement binary search on a sorted array.
- **Difficulty**: Easy
- **Topic**: Searching
- Expect a follow-up on time complexity and why it requires a sorted input.

### Count the frequency of each character in a string.
- **Difficulty**: Easy
- **Topic**: Strings / HashMaps
- Common warm-up; often extended to "find the first non-repeating character."

### Find the longest common prefix among an array of strings.
- **Difficulty**: Medium
- **Topic**: Strings
- Tests careful iteration/edge-case handling (empty array, no common prefix).

### (Power Programmer track) Find the shortest path in an unweighted graph using BFS.
- **Difficulty**: Medium-Hard
- **Topic**: Graphs / BFS
- Reserved for candidates in the harder Power Programmer coding round.

### (Power Programmer track) Solve the 0/1 Knapsack problem.
- **Difficulty**: Hard
- **Topic**: Dynamic Programming
- A strong signal question distinguishing Power Programmer-tier candidates.

### Write a SQL query to find duplicate rows in a table.
- **Difficulty**: Easy-Medium
- **Topic**: SQL
- Expect a `GROUP BY` + `HAVING COUNT(*) > 1` based solution.
