## How Infosys Probes Your Resume

Infosys interviewers generally start from a resume project and use it as a launchpad into broader
technical questions — e.g. if your resume lists a web app, they may ask about the project itself
briefly, then pivot into general questions on the underlying technology (databases, APIs, OOP)
using the project only as context. Power Programmer-track candidates get noticeably deeper
follow-ups, including live coding tied loosely to the project's domain.

Common follow-up angles:
- "Walk me through the architecture of this project."
- "What would you do differently if you rebuilt this today?"
- "How did you handle errors or edge cases in this project?"
- "What was your specific contribution if this was a team project?"

For candidates with AI/ML or modern backend projects (LangChain, FastAPI, cloud-hosted apps),
expect at least one question checking whether you understand the underlying concept (not just
usage) — e.g. "why this tool instead of a simpler alternative?"
