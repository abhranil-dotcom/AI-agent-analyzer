## Candidate Experiences

### Candidate A — Assistant System Engineer (Campus, Digital track)
"I cleared TCS NQT with a decent score in the coding section (2 problems — a string
manipulation one and an array one, both Easy-Medium). The technical interview was about 25
minutes: they asked me to explain OOP concepts using examples from my final-year project (a
Java-based library management system), then moved into basic SQL — I had to write a query to
find duplicate records. They also asked about normalization, which I hadn't revised well and
fumbled a bit, but recovered by walking through an example table live. The HR round was short —
mostly about relocation flexibility and why I wanted to join TCS. Overall the vibe was relaxed,
not adversarial; they seemed to be checking for trainability more than expert knowledge."

### Candidate B — Digital Full Stack Developer (Lateral, 2 years experience)
"Since I was a lateral hire, there was no NQT — just two technical rounds. The first was with a
project lead who dug into my React + Spring Boot experience: REST API design, how I handled
authentication in my last project, and a live coding task to reverse a linked list. The second
round was with a delivery manager, more about domain exposure (I'd worked in BFSI) and how I'd
handle a client escalation. HR discussed compensation band and notice period. The whole process
took about 3 weeks from resume screening to offer."

### Candidate C — Campus fresher, Mainframe track
"I didn't have mainframe experience going in — nobody really does at the fresher level. The
technical interview focused entirely on core CS fundamentals (OS, DBMS) and my C programming
project, not COBOL specifically. The interviewer explicitly said TCS trains freshers on mainframe
technology after onboarding, so they were evaluating general aptitude and communication. HR asked
about willingness to work in a legacy tech stack, which surprises some candidates expecting only
'modern' tech — being open to it clearly helped."

### Candidate D — Cloud Engineer (Lateral, AWS focus)
"Technical round was AWS-heavy: EC2 vs Lambda trade-offs, how I'd design auto-scaling for a
web app, and a scenario question about reducing cloud costs for a client. They also asked basic
Linux commands and a bit of Python scripting. HR was mostly logistical — location, notice period,
and current CTC discussion. The overall tone across both rounds was collaborative rather than
interrogative — they clearly wanted to see how I'd fit into a client-facing delivery team."
