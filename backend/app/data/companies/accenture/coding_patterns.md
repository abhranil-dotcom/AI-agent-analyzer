## Coding Questions

Accenture's coding bar is Easy-to-Medium for general fresher hiring, rising to Medium for
digital-engineering, cloud, and data-track lateral hires, with more emphasis on explaining
trade-offs than at pure IT-services peers.

### Find the intersection of two arrays.
- **Difficulty**: Easy
- **Topic**: Arrays / HashSets
- Expect a follow-up comparing a HashSet approach vs sorting-and-merging.

### Check if a string has all unique characters.
- **Difficulty**: Easy
- **Topic**: Strings
- Simple warm-up; follow-up often asks about doing it without extra data structures.

### Implement a basic LRU cache.
- **Difficulty**: Medium-Hard
- **Topic**: Design / HashMaps / Linked Lists
- A recurring Accenture favorite for digital-engineering-track candidates — tests both data
  structure knowledge and system-design thinking.

### Find the longest substring without repeating characters.
- **Difficulty**: Medium
- **Topic**: Strings / Sliding Window
- Expect discussion of the sliding-window technique and its O(n) complexity.

### Merge overlapping intervals.
- **Difficulty**: Medium
- **Topic**: Arrays / Sorting
- Common in scheduling/calendar-style problem framing.

### Write a SQL query to calculate running totals per customer.
- **Difficulty**: Medium
- **Topic**: SQL / Window Functions
- Expect a `SUM() OVER (PARTITION BY ... ORDER BY ...)` based solution.

### Detect a cycle in a linked list.
- **Difficulty**: Medium
- **Topic**: Linked Lists
- Floyd's cycle detection (slow/fast pointer) is the expected optimal approach.

### Given a list of transactions, find the top-N customers by total spend.
- **Difficulty**: Medium
- **Topic**: Arrays / HashMaps / Sorting
- Framed as a business scenario, reflecting Accenture's consulting-oriented question style.

### Implement a basic rate limiter (conceptually or in pseudocode).
- **Difficulty**: Medium
- **Topic**: Design
- Tests practical system-design thinking for cloud/backend-track candidates.

### Find the missing and duplicate number in an array of 1 to N.
- **Difficulty**: Medium
- **Topic**: Arrays / Math
- Expect a discussion of multiple approaches and their space/time trade-offs.
